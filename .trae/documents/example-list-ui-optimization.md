# ExampleList 页面 UI 优化与交互体验提升方案

## 一、现状分析

当前 ExampleList 页面存在以下问题：

| 维度      | 问题                                                   |
| ------- | ---------------------------------------------------- |
| **布局**  | 使用 CSS `columns-4` 瀑布流，无响应式适配，窗口缩小时 4 列挤压变形          |
| **视觉**  | 封面图无卡片容器，hover 仅 shadow 变化；空状态占位单调；无页面标题区域；工具栏缺乏视觉层级 |
| **交互**  | 无列表项过渡动画；ExampleView 面板展开/收起无动画；分页切换无过渡；无选中态反馈       |
| **骨架屏** | 6 个等高矩形，无差异化，与实际内容布局不匹配                              |
| **响应式** | 固定 4 列，不考虑窗口尺寸变化（最小窗口 800×600）                       |
| **分页**  | 传统分页器，底部控制栏布局拥挤，信息提示按钮与分页器混在一起                       |

## 二、优化方案

### 2.1 布局重构

**目标**：建立清晰的页面结构层级，实现响应式网格布局。

1. **页面结构重组**：分为「工具栏 → 内容网格 → 底部分页 → 详情面板」四个区域，每个区域有明确的视觉边界
2. **响应式网格**：替换 `columns-4` 为 CSS Grid 布局，根据容器宽度自适应列数：

   * 宽度 < 600px：2 列

   * 宽度 600-900px：3 列

   * 宽度 > 900px：4 列
3. **卡片化设计**：每个示例项包裹在卡片容器中，包含封面图 + 底部信息条（显示更新时间或图片数量）
4. **详情面板**：使用 `<Transition>` 组件实现滑入/滑出动画，替代当前的 v-if 硬切换

### 2.2 视觉优化

**目标**：提升视觉吸引力，建立清晰的设计层级。

1. **工具栏优化**：

   * 添加页面标题「示例库」，使用 `text-lg font-semibold` 层级

   * 按钮组右对齐，添加分隔线区分标题与操作区
2. **卡片样式**：

   * 白色背景卡片 + `border` + `rounded-(--radius-lg)` + `shadow-(--shadow-sm)`

   * Hover 时 `shadow-(--shadow-md)` + 轻微上移 `translateY(-2px)` + `transition-all duration-200`

   * 选中态：左侧主色边框 + `bg-(--color-primary-50)` 背景
3. **封面图优化**：

   * 使用 `aspect-ratio: 4/3` 保持统一比例

   * `object-cover` 填充模式

   * 无封面时使用渐变背景 + 居中图标，替代纯灰色
4. **空状态优化**：

   * 增大图标尺寸和间距

   * 添加操作引导文案「点击上方按钮添加第一个示例」

   * 添加淡入动画
5. **骨架屏优化**：

   * 卡片式骨架：封面区域 + 底部文字条区域

   * 交错动画延迟，形成波浪效果

### 2.3 交互优化

**目标**：增强操作反馈，实现平滑过渡效果。

1. **列表项过渡动画**：

   * 使用 `<TransitionGroup>` 实现列表项的进入/离开动画

   * 进入：`fade + slide-up`（从下方淡入）

   * 离开：`fade + scale-down`（缩小淡出）
2. **详情面板过渡**：

   * 使用 `<Transition name="slide-up">` 包裹 ExampleView

   * 面板从底部滑入，收起时向下滑出
3. **分页切换过渡**：

   * 切换页码时，网格内容使用 `fade` 过渡

   * 滚动位置自动回到顶部
4. **Hover 微交互**：

   * 封面图 hover 时添加轻微放大效果 `scale(1.02)`

   * 操作按钮 hover 时添加背景色变化
5. **右键菜单**（增强）：

   * 封面图右键添加上下文菜单（打开画廊、删除示例），替代当前的纯提示文案

### 2.4 分页优化

**目标**：改进分页机制，降低操作门槛。

1. **分页器样式优化**：

   * 居中显示，与内容区有明确间距

   * 使用 `small` 尺寸分页器，减少占用空间
2. **页码信息展示**：

   * 在分页器旁显示「第 X-Y 项，共 Z 项」信息
3. **键盘导航**：

   * 支持 `←/→` 方向键翻页

### 2.5 响应式设计

**目标**：确保在不同窗口尺寸下均有良好显示。

1. **使用 CSS Grid +** **`auto-fill`** **+** **`minmax()`** 实现自适应列数：

   ```css
   grid-template-columns: repeat(auto-fill, minmax(180px, 1fr));
   ```
2. **工具栏响应式**：

   * 窄屏时按钮文字隐藏，仅显示图标
3. **详情面板响应式**：

   * 窄屏时详情面板全宽覆盖而非并排显示

## 五、设计原则

* **一致性**：与项目现有设计系统（base.css 变量体系）保持一致

* **渐进增强**：不改变现有功能逻辑，仅优化视觉和交互

* **性能优先**：动画使用 CSS transform/opacity，避免触发重排

* **可访问性**：保持键盘可操作性，添加合理的 ARIA 属性
